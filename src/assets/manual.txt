
Der MainSim-Cluster ist nach den Vorgaben des Universitätsrechenzentrums nur innerhalb des Campus-Netzwerkes der TU Chemnitz erreichbar.
Um von außerhalb Zugriff zu erhalten, ist eine Anmeldung im VPN (vpn: vpngate.hrz.tu-chemnitz.de) oder auf dem Login-Server (ssh: login.tu-chemnitz.de) notwendig.

Die Home-Verzeichnisse sind per BeeGFS auf allen Rechenknoten verfügbar, Ort: /beegfs-home/users/m.lokamani
Das Verzeichnis kann nur von Ihnen eingesehen werden, Ihnen stehen maximal 1TB zur Verfügung.
Bei Überschreiten der Quota können sämtliche Schreibzugriffe blockiert werden.

Der Skalmod-Gruppe von Prof. Gemming steht ein Austauschverzeichnis unter /beegfs-home/share/skalmod zur Verfügung.

Es wird ein tägliches, wöchentliches und monatliches Backup aller Homeverzeichnisse angelegt.
Das Backup steht in der Regel auf dem Headnode unter /beegfs-home/backup lesend zur Verfügung. 

Als Queueing-System ist SLURM installiert.
Details zur Nutzung erhalten sie per `man sbatch` oder online.
Die Partitionen können auf der oben genannten Statusseite eingesehen werden.
Reservieren sie unbedingt ausreichende Ressourcen für ihre Rechnungen, insbesondere genügend CPU-Kerne, wenn sie parallel rechnen (z.B. sbatch -n 2 -c 24)
Zu hoher Ressourcenverbrauch kann zum Abschuss der Jobs führen.

Verzichten sie unbedingt auf Rechnungen auf dem Headnode.
Hoher CPU-Verbrauch kann zum automatischen Abschuss der Prozesse führen.
Zum Rechnen haben wir die Rechenknoten.

Die Simulationssoftware ist über modulefiles verfügbar.
Liste der verfügbaren Software: `module spider`, laden mit `module load`
Bitte beachten sie, dass einige Softwarepakete nur einen begrenzten Pool an Lizenzen haben.
Unbefugte Benutzung kann zur Sperrung des Accounts führen.

Eigene Software kann im Home-Verzeichnis genutzt werden.
Soll Software für mehrere Nutzer installiert werden, muss diese vorher von ihnen auf dem MainSim erfolgreich kompiliert und getestet werden.
Danach lassen sie mir eine detaillierte Abfolge von Befehlen (besser noch: Shellscript) zukommen, mit der die Administratoren die Software kompilieren können.
Sie kann dann als Module installiert werden.
Bitte unterlassen sie Softwareinstallationen in Austauschverzeichnissen.

Lizenzen können in der Regel auf den Lizenzservern des Clusters installiert werden.
Dafür benötigen wir die Lizenzdatei und Informationen über den notwendigen Lizenzserver.
Eventuell wird eine Installation eines Lizenzservers notwendig, wofür wir die Installationsdatei des Lizenzservers benötigen.

Die GPU-Knoten stehen noch nicht zur Verfügung.
